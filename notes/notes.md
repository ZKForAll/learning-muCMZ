## The prime-order group (O24 §3)

The schemes are built over a group description Γ = (𝔾, p, G) output by
GrGen(1^λ), where 𝔾 has odd prime order p, is generated by G, and has hard
discrete logarithm. The group is written additively and the scalars are ℤ_p.

Prime order p makes 𝔾 cyclic, makes every nonzero element a generator, and makes
x ↦ x • G an isomorphism ℤ_p ≅ 𝔾. Since p is prime, ℤ_p is a field. The discrete
logarithm assumption states that recovering x from X = x • G is infeasible, while
computing x • G is efficient.

In Lean the group setting is fixed as typeclass context, not as a GrGen
procedure. The following binders suffice to define and evaluate the primitives.

```lean
variable (p : ℕ) [Fact p.Prime] {G : Type*} [AddCommGroup G] [Module (ZMod p) G] (𝒢 : G)
```

`Module (ZMod p) G` already forces `p • x = 0`. The facts `[IsAddCyclic G]` and
`Nat.card G = p` are added only when a proof needs the `ZMod p ≃ G` bijection. DL
hardness is a §3.1 experiment, not a typeclass.

The concrete instance is a Schnorr group, the order-p subgroup of (ℤ_q)ˣ with p
dividing q − 1, using toy parameters p = 11, q = 23, 𝒢 = 4. It is exactly prime
order and computable. An elliptic curve such as Ristretto255 is the production
alternative and a heavier instantiation, and the abstract primitives are
identical over either.
